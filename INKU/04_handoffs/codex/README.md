# Codex Execution Handoffs

Ce dossier reçoit uniquement les prompts d’exécution Codex validés pour les phases actives du projet INKU.

## Règles

- un fichier = un mandat d’exécution clair
- un prompt principal Omega = une phase ou un objectif d’implémentation précis
- aucun prompt ici ne doit contredire :
  - `01_preproduction/00_master/`
  - `04_handoffs/implementation_briefs/`
- les prompts Codex doivent exécuter, pas redéfinir le produit
- aucun prompt ne doit élargir le scope sans validation explicite en amont

## Ordre d’autorité

1. `01_preproduction/00_master/`
2. `04_handoffs/implementation_briefs/`
3. fichiers du présent dossier

## Convention de nommage recommandée

- `OMEGA_CODEX_PHASE_BC_EXECUTION.md`
- `OMEGA_CODEX_PHASE_D_...md`
- `OMEGA_CODEX_PHASE_E_...md`

## Règle de discipline

Ce dossier ne doit pas devenir une archive de prompts brouillons.
Il doit contenir uniquement des handoffs Codex propres, versionnés, exploitables, et alignés avec les documents autoritatifs du projet.
