# INKU V1 — Slots

## Locked rule
V1 has exactly **4 persistent slots**.

## Product role
Each slot is:
- a container of ownership
- a passive-income source
- a collection unit
- a management point
- a persistence anchor

## Core rules
- 1 slot = 1 slime
- purchase auto-fills first free slot
- if all slots are full, purchase is refused
- slot order matters
- drag swaps slot positions

## Visible slot data
- slime miniature
- name
- rarity
- income/min
- one visible synthetic state

## Slot interactions
### Short tap
- focus / selection

### Drag
- reorder only

### Long press
- open profile sheet

## Living feel
Stored slimes must not feel dead.
Slot presentation should preserve:
- light animation or pulse
- persistent visible mood cue
- continuing income identity
