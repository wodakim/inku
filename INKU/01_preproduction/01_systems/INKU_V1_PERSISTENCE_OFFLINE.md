# INKU V1 — Offline Persistence

## Persistence goal
A bought slime must remain the same slime across sessions:
- same identity
- same slot
- same name
- same economic value
- same main states

## Save categories
### Persist exactly
- money
- slots and order
- slime records
- seed
- visual genome
- rarity
- purchase price
- income/min
- resale value
- stable traits
- main dynamic states
- timestamps
- current incubator offer + expiry
- save version

### Recompute
- synthetic visible labels
- profile phrase
- prairie transient micro-state
- UI summaries

### Do not persist as exact runtime state
- current animation frame
- instantaneous velocity
- current drag state
- tiny temporary movement intentions

## Offline income
Use:
- last passive tick timestamp
- time delta on return
- income generated by owned slot slimes
- simple bounded design extensible later

## Prairie persistence
Persist:
- whether slime is marked in prairie

Do not persist:
- frame-perfect position or transient motion state

## Integrity
Need simple cleanup strategy for:
- broken slot references
- duplicate slime references
- invalid data
