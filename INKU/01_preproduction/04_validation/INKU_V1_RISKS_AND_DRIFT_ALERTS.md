# INKU V1 — Risks and Drift Alerts

## Main risks
- system complexity explosion
- unreadable mobile UI
- fake depth with invisible variables
- slots becoming dead inventory
- prairie becoming decorative only
- economy becoming generic idle sludge
- persistence incoherence

## Drift alerts
Project is drifting if:
- slot UI density grows
- many new stats are added
- prairie becomes optimization-heavy
- social depth appears before V1 proof
- backend complexity arrives before local loop validation

## Countermeasures
- cap systems aggressively
- require visible payoff for every variable
- preserve mobile readability
- test loop truth before adding ambition
